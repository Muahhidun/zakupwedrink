# WeDrink Stock Manager Bot

Telegram бот для учета складских остатков и формирования закупок ингредиентов WeDrink.

## 🚀 Быстрый старт

### 1. Установка зависимостей

```bash
cd /Users/Dom/projects/wedrink-bot
python3 -m venv venv
source venv/bin/activate
pip install -r requirements.txt
```

### 2. Настройка

Создайте файл `.env` на основе `.env.example`:

```bash
cp .env.example .env
```

Заполните обязательные параметры:

1. **BOT_TOKEN**: Получите у @BotFather
2. **REMINDER_CHAT_ID**: ID группы для ежедневных напоминаний
   - Добавьте бота @userinfobot в вашу группу
   - Напишите любое сообщение в группу
   - Бот покажет Chat ID (например: -1001234567890)
   - Удалите @userinfobot из группы

Пример `.env`:
```env
BOT_TOKEN=7894561230:AAHdqTcvCH1vGWJxfSeofSAs0K5PALDsaw
REMINDER_CHAT_ID=-1001234567890
DATABASE_PATH=wedrink.db
```

### 3. Импорт данных из CSV

Первый запуск - импортируем товары и остатки из таблицы:

```bash
python import_data.py
```

### 4. Запуск бота локально

```bash
python main.py
```


### 2. Создать репозиторий на GitHub

```bash
# Создайте репозиторий на github.com
# Затем:
git remote add origin https://github.com/YOUR_USERNAME/wedrink-bot.git
git branch -M main
git push -u origin main
```

### 3. Деплой на Railway

1. Зайдите на railway.app
2. New Project → Deploy from GitHub repo
3. Выберите `wedrink-bot`
4. Railway автоматически обнаружит `Procfile`
5. Добавьте переменные окружения:
   - `BOT_TOKEN` = ваш токен
   - `ADMIN_IDS` = ID пользователей

**⚠️ ВАЖНО:** Перед деплоем на Railway:

1. Запустите `python import_data.py` локально
2. После импорта скопируйте файл `wedrink.db` в Railway (через Railway CLI или Volume)

Или добавьте команду в `Procfile`:

```
release: python import_data.py
worker: python main.py
```

## 📱 Использование бота

### Команды

**Склад:**
- `/stock` - Ввести остатки на сегодня
  - **Новинка:** Выбор между массовым вводом (все товары списком) и последовательным
  - При массовом вводе: введите количество упаковок через пробел или запятую
  - После сохранения показывается мини-отчет с цветовой индикацией:
    - 🔴 Красный: товара нет в наличии
    - 🟠 Оранжевый: хватает до 3 дней (критический запас)
    - 🟡 Желтый: хватает 4-7 дней (низкий запас)
    - 🟢 Зеленый: хватает больше недели
- `/current` - Текущие остатки

**Заказы:**
- `/order` - Список товаров для закупа (10 дней)
- `/order7` - Заказ на 7 дней
- `/order14` - Заказ на 14 дней

**Отчеты:**
- `/report` - Расход за вчера в ₸
- `/week` - Расход за неделю
- `/analytics` - Аналитика по товарам

### Автоматические напоминания

**⏰ Ежедневное напоминание в 11:00**
- Бот автоматически отправляет напоминание о добавлении остатков в группу
- Настраивается через переменную `REMINDER_CHAT_ID` в `.env`
- Часовой пояс: Asia/Almaty (UTC+6)

### Workflow

1. **11:00 каждый день:** Бот отправляет напоминание в группу
2. **Ввод остатков:** Администратор нажимает `/stock` и выбирает способ ввода:
   - **Массовый (рекомендуется):** Все товары вводятся одной строкой
   - **Последовательный:** По одному товару
3. **Мини-отчет:** После ввода бот показывает критические товары с цветовой индикацией
4. **При поставке:** `/supply` (будет добавлено позже)
5. **Планирование закупа:** `/order` - смотрим что заказать
6. **Отчеты:** `/report` или `/week` - расходы в деньгах

## 🐛 Troubleshooting

**Бот не отвечает:**
- Проверьте токен в `.env`
- Убедитесь что бот запущен (`python main.py`)

**Ошибка импорта:**
- Проверьте путь к CSV файлу в `import_data.py`
- Убедитесь что CSV в UTF-8

**Пустой список заказа:**
- Нужна история остатков минимум за 2-3 дня
- Введите остатки через `/stock` несколько раз
